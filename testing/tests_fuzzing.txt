Тест-кейс №1
	Название:
		Передача странице несуществующего параметра
	Предусловия:
		1. Открыт браузер
		2. Существует страница http://localhost:8080/MuseStore/answer_aut.jsp, принимающая параметры login и password
		3. Шаблон запроса: http://localhost:8080/MuseStore/answer_aut.jsp?parameter=some_value
	Шаги:
		1. Кликнуть по адресной строке браузера
		2. Ввести «http://localhost:8080/MuseStore/answer_aut.jsp?»
		3. После знака «?» прописать любое имя параметра, знак «=», любое значение параметра(см. шаблон запроса в предусловии)
		4. Нажать на «Enter»
	Ожидаемый результат: 
		Запрашиваемый ресурс недоступен
		
Тест-кейс №2
    Название:
      	Автоматическое исправление типа пользователя при некорректном указании значения в строке запроса для параметра user_type
	Предусловия:
		1. Открыт браузер
		2. Существует страница http://localhost:8080/MuseStore/registration.html с параметрами login, password, name, email, type
		3. Существует страница http://localhost:8080/MuseStore/answer_reg.jsp, принимающая параметры login, password, name, email, type
		4. Корректным значением в поле user_type является 0 или 1
		5. Шаблон запроса: http://localhost:8080/MuseStore/answer_reg.jsp?login=user&password=user&name=Ivanov&email=user@gmail.com
	Шаги:
		1. Кликнуть по адресной строке браузера
		2. Ввести «http://localhost:8080/MuseStore/answer_reg.jsp?»
		3. После знака «?» присвоить корректное значение(см. документацию)  параметрам login, password, name, email в соответствии с шаблоном в предусловии
		4. Присвоить параметру user_type значение, содержащее цифру и букву
	Ожидаемый результат: 
		Форма, сообщающая об успешной регистрации. Значение user_type  пользователя автоматически установлено в 1.

Тест-кейс №3
	Название:
		Проверка создания пользователей с одинаковым логином
	Предусловия:
		1. Открыта страница http://localhost:8080/MuseStore/registration.html
		2. Корректно(см. документацию) заполнены поля «Пароль», «ФИО», «email»
		3. Существует пользователь с логином user
	Шаги:
		1. Ввести в поле «Логин» значение «user»
		2. Нажать на кнопку «Зарегистрировать»
		3. Повторить шаги 1-2 с различным заполнением поля «Пароль»
	Ожидаемый результат: 
		Сообщение об ошибке создания пользователя с логином user
